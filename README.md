# koishi-plugin-wifegacha-next

> 抽老婆插件 | **开发中...**

一个用于 Koishi 的群聊娱乐插件。
功能设计来源于 [WifeGacha](https://github.com/Rlezzo/WifeGacha/tree/master)、[Bear-biscuit/koishi-plugin-wifegacha](https://github.com/Bear-biscuit/koishi-plugin-wifegacha) 与 [magisk317/koishi-plugin-wifegacha](https://github.com/magisk317/koishi-plugin-wifegacha)。本仓库在前人工作基础上继续维护，非常感谢原作者们的付出。

## 新增增强功能

- **好感度系统**：`日老婆`、`亲老婆/亲亲`、`约会` 会影响当前老婆好感度，并支持可配置彩蛋事件。
- **个人与群档案**：`用户档案` 只展示当前群内个人数据，`群档案` 展示当前群聚合统计。
- **查老婆增强**：展示当前老婆、来源、图片、当前好感度和互动冷却状态。
- **群老婆档案**：查询指定老婆在当前群的统计，不填名称时展示本群老婆榜单。
- **老婆图鉴优化**：缩略图会预生成并缓存在 `data/assets` 下，缺失时自动生成，减少图鉴生成负载。
- **全局消息撤回面板**：各类指令可独立配置是否撤回插件回复。
- **管理命令优化**：新增老婆采用分步向导，更新/删除/重命名老婆会同步刷新图鉴缓存。

## 已实现功能

- **抽老婆**：随机抽取一个老婆。
- **牛老婆**：尝试牛走群友的当前老婆，成功概率会参考双方好感度与牛老婆行为。
- **日老婆**：与当前老婆互动并改变好感度。
- **亲老婆 / 亲亲**：短冷却、低收益的好感互动。
- **约会**：长冷却、高随机性的好感互动。
- **交换老婆**：与他人互换当前老婆，并同步好感状态。
- **离婚**：解除与当前老婆的绑定关系。
- **查老婆**：查看自己或被 @ 群友的当前老婆、来源、图片、好感度和冷却信息。
- **老婆图鉴**：查看自己或被 @ 群友在当前群的老婆收集图鉴。
- **用户档案**：查看当前群内个人抽取、离婚、牛老婆、好感等数据。
- **群档案**：查看当前群的抽取、牛老婆、离婚、交换、互动统计。
- **群老婆档案**：查看当前群老婆榜单或指定老婆统计。
- **新增老婆**：通过分步向导向老婆池添加新老婆。
- **更新老婆**：管理员更新指定老婆图片并刷新缓存。
- **删除老婆**：管理员删除指定老婆数据、图片和相关缓存。
- **重命名老婆**：管理员按命名规则重命名老婆并同步资源文件名。
- **更新老婆数据**：重新同步老婆资源目录，适用于手动增删图片后的刷新。

## 使用前准备
- 下载[老婆包](https://github.com/Bear-biscuit/koishi-plugin-wifegacha/releases)
- 放在 `data/assets/wifegacha/` 目录中。
- 命名规则：`来源+名称`
```
wifegacha/
├── 孤独摇滚+后藤一里.png
├── 孤独摇滚+山田凉.png
└── 孤独摇滚+伊地知虹夏.png
```


## 从 koishi-plugin-wifegacha-enhanced 迁移

如果你此前使用的是 `koishi-plugin-wifegacha-enhanced`，迁移到本包时建议：

1. 备份 Koishi 数据库和 `data/assets/wifegacha`。
2. 在 Koishi 控制台禁用旧插件。
3. 安装并启用 `koishi-plugin-wifegacha-next`。
4. 手动复制旧插件配置。
5. 重启 Koishi，并验证 `抽老婆`、`查老婆`、`日老婆`、`亲老婆`、`约会`、`牛老婆`、`老婆图鉴`。

本包继续使用 `wifeUser`、`wifeData`、`groupData`、`wifeMigration` 等数据表，正常情况下可以复用旧数据。不要同时启用旧包和新包，否则可能出现命令冲突或数据竞争。
